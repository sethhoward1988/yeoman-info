Deploying with Grunt

Grunt is a powerful automation tool for web development. It can compile, minify, unit test, and lint your assets and be customized with other tasks helpful in your specific workflow and development style.

Grunt tasks are defined in the Gruntfile.js. Yeoman comes pre-defined with several tasks that are very important in Yeoman workflow: (a) Grunt Build, (b) Grunt Server, and (c) Grunt Test.

Grunt Build

The Build task is a super-task which triggers several other sub-tasks such as clean, cssmin, uglify, copy, htmlmin, etc. You can see all the tasks that build triggers by checking your Gruntfile.js. The following is a section from a default Gruntfile.js: 

              grunt.registerTask('build', [
                  'clean:dist',
                  'useminPrepare',
                  'concurrent:dist',
                  'autoprefixer',
                  'concat',
                  'cssmin',
                  'uglify',
                  'copy:dist',
                  'rev',
                  'usemin',
                  'htmlmin'
              ]);

In essence, the grunt build command will clean up your directories, minify and concatentate assets (HTML, CSS, JS, Images), and rename and move generated files to the dist/ directory.

Grunt Server

Yeoman pulls in a handy grunt task grunt serve which will actually fire up a development server so you can run your code in a web browser. The following is a snippet from a default Gruntfile.js showing the serve task. 

              grunt.registerTask('serve', function (target) {
                  if (target === 'dist') {
                      return grunt.task.run(['build', 'connect:dist:keepalive']);
                  }

                  grunt.task.run([
                      'clean:server',
                      'concurrent:server',
                      'autoprefixer',
                      'connect:livereload',
                      'watch'
                  ]);
              });

A handy part of the Grunt server is that it will watch and reload any assets that you edit.

Grunt Test

Testing your code is an important step in creating code safe for production. Yeoman comes with a task that will run tests for your dependencies and custom code: grunt test.

Unit tests are defined in test/spec/test.js. Grunt uses Mocha and Chai.js as its testing system (which you can see in test/lib/). The following is a snippet from the test.js file: 

              /*global describe, it */
              'use strict';
              (function () {
                  describe('Give it some context', function () {
                      describe('maybe a bit more context here', function () {
                          it('should run here few assertions', function () {

                          });
                      });
                  }); 
              })();

Read up on Mocha and Chai.js to learn more about writing JavaScript unit tests.
