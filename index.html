Discovering Yeoman

Yeoman is a command-line tool that helps web developers work efficiently by automating workflow processes. Yeoman is actually a package of several tools: Yo, Bower, and Grunt which respectively (a) scaffold project layouts, (b) manage dependencies, and (c) automate the build/deploy process.

This article will briefly introduce the three components of Yeoman. Additional information on each individual tool can be found on the other pages of this site.

Yo Scaffolding

Scaffolding is handled by a command-line script called yo. It is unique to the Yeoman project and is installed using npm. The following code with install yo with all its components: 

              $ npm install -g yo

With Yo you can scaffold out a folder structure for an entire project or generate specific files from defined templates.

Yo operates on pre-defined templates called generators. A generator is installed using npm or by writing one yourself. The following are some useful generators in the Yeoman community:

  • webapp
  • angular
  • wordpress
  • backbone
  • ember

Bower Dependencies

Bower is an open-source package manager for web resources. It can manage assets such as JavaScript, CSS, and Images. Bower has many of the capabilities found in a package manager such as searching, installing, listing, updating, and uninstalling assets. The following snippets show bower in action: 

              # Search for a dependency in the Bower registry.
              bower search dep

              # Install one or more dependencies.
              bower install dep depN

              # List out the dependencies you have installed for a project.
              bower list

              # Update a dependency to the latest version available.
              bower update dep

Grunt Deploy

Grunt handles the building and deploying of your project but it is capable of much more. Grunt is a JavaScript task runner and can handle pretty much any kind of automation including running unit tests, compiling CSS pre-processors (e.g. SASS or LESS), compiling CoffeeScript, minifiying static assets, or previewing your app on a dev server.

Now What?
